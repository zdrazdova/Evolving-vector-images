class triangulation.Line_segment(a, b)¶

Class representing one line segment. Line segment is determined by two endpoints.

not_crossing(lines)¶

Check whether line_segment intersects with any line_segment from list lines. Method goes through the list and for each item calculates determinant for lines corresponding to this item and line_segment. If determinant differs from 0, coordinates of intersection are calculated. If coordinates lie somewhere on line segments excluding their endpoints, method returns False. Othewise we go to the next item. If all items in the list were checked and no intersection found, method returns True.

Parameters:lines – A list of line segments.
Returns:True/False

class triangulation.Triangle(s, t, u, image)¶

Class representing one triangle. Triangle is determined by three points and a color value.

choose_color(image)¶

Calculate color for a triangle. First, order_points method is used to sort vertices of a triangle in a desired way. To determine which pixels lie in a triangle we start from the top vertex and move in rows downwards. For each row, x coordinates of left and right border are calculated and we work with pixels in between. This process continues until it reaches y coordinates of middle point. Similar process starts in the bottom vertex and moves upwards until it reaches the middle vertex. This way all pixels are recorded. Then for each pixel in a triangle red, green and blue values are extracted. All values for red are added up and and the sum is divided by the total number of pixels in the triangle. Average values for green and blue are calculated in the same manner. Triplet of average values is desired color.

Parameters:image – Source image.
Returns:Triplet of values = color.

triangulation.evolution(image, width, height, pixels, number_of_points, population_size, number_of_generations, vertex_mutation_prob, object_vertex_mutation_prob, border_width, color_distance)¶

Execute evolution. Fitness function is registered. Population of population_size individuals is created. Tools and selection method are registred into toolbox. Simple hall of fame method is implemented. For each generation: Offspring are chosen via selection method. Then mutation is executed and fitness is evaluated. New population is selected from individuals in last population. Hall of fame is updated.

Ten times during the evolution the best individual at that point is saved as a set of rectangles and a set of connected polygons in SVG files. After all generations, the best individual is saved as a set of rectangles and a set of connected polygons in SVG files.

Parameters:
  • width – Width of source image.
  • height – Height of source image.
  • relevant_colors – List of relevant colors according to frequency.
  • pixels – List of color values of pixels in source image.
  • number_of_objects – Initial number of shapes in one individual.
  • population_size – Number of individuals in a population.
  • number_of_generations – Total number of generations to be evolved.
  • color_mutation_prob – Probability that color mutation will be done.
  • object_color_mutation_prob – Probability that color mutation will be executed for an object.
  • dimensions_mutation_prob – Probability that dimensions mutation will be done.
  • object_dimensions_mutation_prob – Probability that dimensions mutation will be executed for an object.
  • position_mutation_prob – Probability that position mutation will be done.
  • object_position_mutation_prob – Probability that position mutation will be executed for an object.
  • xover_prob – Probability that crossover will be done.

triangulation.generate_points(width, height)¶

Generate number_of_points points, for each point, x and y coordinates are saved. Points are placed on a canvas width dimensions width and height. First, four points are placed in the corners of the canvas. Then canvas is divided into parts of equal area and one point is placed randomly within one area. The rest of the points is placed randomly anywhere as long as none two points have the exact same coordinates. Method already_exists of class Point is used to check whether point with identical coordinates already exists.

Parameters:
  • width – Width of the canvas.
  • height – Height of the canvas.
Returns:

A list of points.

multi_objective.evolution(width, height, relevant_colors, pixels, number_of_objects, population_size, number_of_generations, color_mutation_prob, object_color_mutation_prob, dimensions_mutation_prob, object_dimensions_mutation_prob, position_mutation_prob, object_position_mutation_prob, add_prob, delete_prob, xover_prob, border_width)¶

Execute evolution. Fitness function is registered. Population of population_size individuals is created. Tools are registred into toolbox. Hall of fame method - Pareto front - is implemented. For each generation: Offspring are chosen via TournamentDCD selection. Then crossover and mutations are executed and fitness is evaluated. New population is selected from individuals in last population and offspring. Hall of fame is updated and fitness is saved to log. After all generations, unique individuals from Pareto front are chosen and saved as SVG files. Log of their fitnesses is created.

Parameters:
  • width – Width of source image.
  • height – Height of source image.
  • relevant_colors – List of relevant colors according to frequency.
  • pixels – List of color values of pixels in source image.
  • number_of_objects – Initial number of shapes in one individual.
  • population_size – Number of individuals in a population.
  • number_of_generations – Total number of generations to be evolved.
  • color_mutation_prob – Probability that color mutation will be done.
  • object_color_mutation_prob – Probability that color mutation will be executed for an object.
  • dimensions_mutation_prob – Probability that dimensions mutation will be done.
  • object_dimensions_mutation_prob – Probability that dimensions mutation will be executed for an object.
  • position_mutation_prob – Probability that position mutation will be done.
  • object_position_mutation_prob – Probability that position mutation will be executed for an object.
  • add_prob – Probability that add_object mutation will be done.
  • delete_prob – Probability that delete_object mutation will be done.
  • xover_prob – Probability that crossover will be done.
